Jenkins & Production

I won't spend a ton of time here but I wanted to give an overview and let you know how the production systems are working. We are using software called Jenkins, which is a Continous Integration Software package, (aka CI) to watch for changes to the code repositories. As soon as Jenkins sees a change, it will pull the code and automatically update the "production" environment with it. Jenkins is looking for those changes every 2 minutes.

When you type `git push` you will change production. Hint: It's wise to ask your team leader the first couple of times before you push.

The production version of your code will be at the following locations:

Application Production URL
Website http://web.pyatt.lan
CRM Portal http://crm.pyatt.lan
CRM API http://crmapi.pyatt.lan
Ops Portal http://ops.pyatt.lan
Ops API http://opsapi.pyatt.lan
Mobile Ask the Patels
Mobile API http://mobileapi.pyatt.lan

We are combining at least two concepts here (QA and Production). Normally you'd have a few environments and not just Dev and Production. In this case, it's HackMT and who are we kidding, we will just be happy if it works and we can demo it.
